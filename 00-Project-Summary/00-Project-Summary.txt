# Project Summary

## Hybrid Environment Foundations And Management

This project demonstrates the build, validation, and troubleshooting of a hybrid environment integrating on-prem Active Directory with Microsoft Entra ID. The focus was on how systems interact across both environments, how identity and management boundaries are defined, and how to troubleshoot issues that occur between on-prem and cloud systems.

The lab environment used a local Active Directory domain (corp.test) connected to a Microsoft Entra tenant using the default onmicrosoft.com domain. This enabled direct comparison between on-prem and cloud representations of users and devices.

The project examined how synchronized users appear in Entra while remaining managed on-prem, and how cloud-only users differ in behavior and management. This was used to determine source of authority and make correct decisions about where changes must be applied.

System consistency was validated by comparing user attributes across Active Directory and Entra, including differences in User Principal Name (UPN) formats. These differences were used to simulate real-world sign-in issues and guide troubleshooting decisions.

The project also included device validation using a domain-joined workstation (W11VM). Device state was verified using command-line tools to determine whether the system was domain joined, Entra joined, or operating within a hybrid configuration.

Throughout the project, findings were documented using an evidence-first workflow, including build validation, troubleshooting evidence, structured tickets, and reusable runbooks aligned to real IT support practices.

Overall, this project demonstrates the ability to work across on-prem and cloud systems, determine management boundaries, validate system behavior, and troubleshoot issues in a hybrid environment.
